# Описание проблемных мест
- построение отчётов занимает много времени
- высокое time-to-market
- сложность интеграции новых бизнесов, большое количество бизнес логики(скорее всего запутанной) в DWH и при интеграции новых бизнесов её нужно будет ещё больше усложнять, сложно масштабируется
- устаревший инструмент Power Builder для взаимодействия с DWH
- много разнообразных видов бизнеса, каждый из которых образует свою экспертную область, но данные хранятся в одном DWH
- много кастомизаций BI-системы поверх DWH
- шина данных на основе Apache Camel требует обновления и сложно масштабируется, что может создать проблемы при росте бизнеса
- DWH на основе старой версии Microsoft SQL-сервера 2008 года не справляется с объёмом данных
